Vibe Coding 执行路线 · 2026-06-14

AI 简历作品集完整项目开发计划

这份计划用于把当前静态设计稿推进成完整可上线项目：React 前台、React 后台管理、FastAPI 后端、MySQL 数据、Dify SSE 流式对话、媒体资源和简历下载。重点不是一次性让 AI 写完所有代码，而是按阶段给 AI 明确上下文、明确产出、逐步验收。

当前状态 已有 PRD、API 规格、MySQL 规格、HTML 设计稿；后台管理已纳入 MVP
目标版本 MVP v1.0，可公开访问和面试展示
技术栈 React + Vite + FastAPI + MySQL + Dify + 后台 CRUD
核心验收 打开首屏即可 AI 提问，后台可维护项目和媒体

1. 你的整体操作流程

  1. 先准备素材和 Dify 应用，不急着写代码。
  2. 让 AI 搭建项目骨架，前后端分开跑通。
  3. 先做后端健康检查、项目接口、简历下载和后台登录。
  4. 实现后台项目、截图、视频、简历、站点配置和预设问题 CRUD。
  5. 再接 Dify API 代理，确认 SSE 流式返回真实可用。
  6. 把静态设计稿拆成 React 前台组件，同时实现后台管理页面。
  7. 做移动端、错误态、加载态、空状态、后台鉴权和安全检查。
  8. 部署上线，最后用真实浏览器做一次完整面试官路径和管理员维护路径测试。
Vibe Coding 原则： 每次只让 AI 做一个阶段，不要一句话说“帮我做完整项目”。你要像产品经理和技术负责人一样，给它输入现有文档、目标、边界、验收标准，然后让它输出代码和自测结果。

Phase 4：后端对接 Dify API 和 SSE 流式输出

目标

实现真正的 AI 智能对话能力。前端永远只请求你自己的后端，不能直接请求 Dify。

安全红线： 真实 DIFY_API_KEY 只能出现在后端服务器环境变量里，不能写进前端、HTML、README 示例、截图或提交记录。

后端环境变量

DIFY_API_BASE_URL=https://api.dify.ai/v1
DIFY_API_KEY=<your-dify-api-key>
DIFY_USER_PREFIX=portfolio-web
CHAT_LOG_ENABLED=true

让 AI 做什么

  • 实现 POST /api/chat/stream。
  • 后端用 httpx.AsyncClient 请求 Dify Chat API。
  • 请求体使用 response_mode: "streaming"。
  • 后端把 Dify 返回的流转成标准 SSE：event: message、event: done、event: error。
  • 保存 conversation_id，支持连续追问。
  • Dify 报错时，返回 event: error，不要让连接卡住。

curl 验收命令

curl -N -X POST http://localhost:8000/api/chat/stream ^
  -H "Content-Type: application/json" ^
  -H "Accept: text/event-stream" ^
  -d "{\"message\":\"请介绍你的 Dify 简历助手项目\",\"visitor_id\":\"local-test\"}"

验收

  • 能看到至少一个 event: message。
  • 最后能看到 event: done。
  • 断网或密钥错误时能看到 event: error。
  • 前端代码和浏览器 Network 里看不到 Dify Key。

给 Vibe Coding 的提示词

请实现 FastAPI 的 Dify SSE 后端代理接口 POST /api/chat/stream。
要求：
1. 前端只请求 /api/chat/stream，不直接请求 Dify
2. DIFY_API_KEY 从环境变量读取
3. 请求 Dify 时 response_mode 使用 streaming
4. 把 Dify 流式响应转换成标准 SSE
5. 输出 event: message / event: done / event: error
6. conversation_id 要能返回给前端继续追问
7. 写一个测试或 mock，验证没有真实 Dify 时也能测试 SSE 格式

Phase 7：部署上线

推荐部署方案

简单方案

  • 前端：Vercel / Netlify / 静态服务器
  • 后端：云服务器 + FastAPI + Uvicorn/Gunicorn
  • 数据库：云 MySQL 或服务器自建 MySQL
  • 媒体：服务器 uploads/

更稳方案

  • 前端：Vercel 或 Cloudflare Pages
  • 后端：Docker 部署 FastAPI
  • 数据库：托管 MySQL
  • 媒体：对象存储 + CDN

上线前必须设置

  • 生产环境 DIFY_API_KEY。
  • 生产环境 MySQL 用户和密码。
  • CORS 只允许你的前端域名。
  • 上传目录不可执行脚本。
  • HTTPS 证书可用。
  • 后端日志能看到 Dify 调用失败原因，但不打印密钥。

上线验收

  • 公网访问首页正常。
  • 手机浏览器打开正常。
  • AI 对话首问成功。
  • 项目截图和视频可加载。
  • 简历可下载。
  • 查看网页源代码没有任何密钥。

4. 每次 Vibe Coding 的标准工作循环

  1. 给上下文：告诉 AI 当前阶段、相关文档、现有目录、不能做什么。
  2. 让它先复述计划：确认它理解任务后再写代码。
  3. 小步实现：一次只做一个接口、一个组件或一个联调点。
  4. 马上验收：运行命令、打开页面、截图、curl 接口。
  5. 修复问题：把错误日志原样给 AI，让它定位，不要让它猜。
  6. 提交记录：每个阶段完成后 commit，避免后面改崩无法回退。
